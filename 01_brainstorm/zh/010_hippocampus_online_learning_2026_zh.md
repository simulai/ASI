## 核心问题

现有方案的问题：

| 方案 | 能否推理时学习？ | 代价 |
|------|----------------|------|
| Transformer（标准）| ❌ 权重冻结 | 无在线学习 |
| Hopfield 网络 | ❌ 训练一次，推理时只读不写 | 无增量学习 |
| KV-Cache | ⚠️ 只缓存，不学习新模式 | 无法泛化 |
| **海马体本身** | ✅ 全可在推理时完成 | — |

我们需要的：**推理时在线学习 + 存储新记忆 + 抗遗忘 + 热力学高效**

## 2026年最新方案

### 方案1：HiPPO Zoo ⭐ 最相关（2026年2月）

**论文**：Goffinet, Hanks & Carlson (2026) — *HiPPO Zoo: Explicit Memory Mechanisms for Interpretable State Space Models*
**arXiv**: `2602.21340`

**这是目前最直接相关的论文。**

核心思路：HiPPO 框架（Gu & Dao 等人提出）通过正交多项式基压缩序列历史，但一直以来 memory 机制是隐式的。HiPPO Zoo 把它显式化：

- **自适应记忆分配**（adaptive memory allocation）：不预设记忆容量，动态分配
- **联想记忆**（associative memory）：给定部分输入，检索完整记忆模式
- **在线训练**：在流式设置下高效训练，权重实时更新
- **可解释**：记忆结构是显式的，不再是黑箱

**和海马体的对应**：
- HiPPO 的多项式基 ≈ 海马体的**正交化表征**（每个 context 都有独立基向量）
- 自适应分配 ≈ 海马体的**场景特异性**（每次新场景激活新细胞群）
- 联想记忆 ≈ 海马体的 **pattern completion**（部分输入 → 完整记忆）

**为什么重要**：
> 这是第一篇把 SSM（状态空间模型）的记忆机制**显式化、可解释化**的论文，而且支持在线学习。相当于把 Mamba 升级成了"带海马体记忆的 Mamba"。

### 方案2：Memento — 案例记忆在线RL（2025年8月）

**论文**：Zhou et al. (2025) — *Memento: Fine-tuning LLM Agents without Fine-tuning LLMs*
**arXiv**: `2508.16153`

核心设计：**Memory-augmented Markov Decision Process (M-MDP)**

- **案例记忆**（episodic memory）：存储过往经验，可以是可微分的或非参数的
- **神经案例选择策略**（neural case-selection policy）：学会从记忆中选择最相关的案例
- **策略改进通过记忆检索**：不需要梯度更新，实现持续实时学习
- **外分布任务提升 4.7–9.6%**

**性能数据**：
- GAIA top-1：87.88% Pass@3
- DeepResearcher：66.6% F1，80.4% PM
- **优于训练-based 方法**，且不需要微调底层 LLM

**为什么重要**：
> 这是真正实现了"推理时学习"的 LLM Agent——不改变模型权重，只通过案例记忆实现持续适应。和海马体的**情景记忆 + 经验驱动适应**完全对应。

### 方案3：PhiNets — CA3/CA1 双预测器架构（2024-2025）

**论文**：Ishikawa et al. (2024-2025) — *PhiNets: Brain-inspired Non-contrastive Learning Based on Temporal Prediction Hypothesis*
**arXiv**: `2405.14650`

核心设计：模拟海马体 **CA3 → CA1** 双路径：

- **CA3 预测器**（类似自编码器的压缩表征）≈ 海马体 CA3（联想记忆）
- **CA1 预测器**（保留时序细节）≈ 海马体 CA1（输出到皮层）
- **X-PhiNet**：带 momentum encoder，在线适应新模式更快
- **持续学习 + 在线学习**：比标准 SimSiam 快适应新模式

**和海马体的直接对应**：
- CA3 的压缩 ≈ Hopfield 吸引子的低能态
- CA1 的时序保真 ≈ 记忆的时间顺序
- 双预测器结构 ≈ 海马体两条并行路径（direct perforant path + trisynaptic circuit）

### 方案4：Online-LoRA / TreeLoRA — 持续学习的参数高效方案（2025）

**论文**：
- Online-LoRA (`arXiv:2411.05663`) — 免任务的在线持续学习
- TreeLoRA (`arXiv:2506.10355`) — 分层梯度相似性引导的 LoRA

核心思路：**不修改原始权重，只更新轻量适配器**

- 灾难性遗忘的最大问题：全参数微调会覆盖旧知识
- LoRA 的解法：冻结原始权重，只训练低秩适配器
- Online-LoRA 进一步：不需要预先知道任务边界，在线检测并适应
- TreeLoRA：用分层梯度树指导哪些 LoRA 该更新、该保留

**对海马体的启示**：
- LoRA ≈ 海马体的**突触可塑性**（只强化特定连接，不改变整体结构）
- 在线 LoRA ≈ 海马体的**快速突触强化**（CA3 的单次学习）
- 树结构 ≈ 海马体的**组织层级**（从 Dentate Gyrus → CA3 → CA1）

### 方案5：DMA — 在线 RAG 对齐（2025年11月）

**论文**：Bai et al. (2025) — *DMA: Online RAG Alignment with Human Feedback*
**arXiv**: `2511.04880`

核心设计：**在线 RAG + 人类反馈闭环**

- 不离线训练，而是**实时接收人类反馈**更新检索策略
- 检索质量 = 用户反馈信号
- 在线优化检索 + 生成对齐

**为什么重要**：
> 这是把"海马体的导航反馈信号"引入 RAG 的实现——海马体通过位置细胞反馈持续校准空间表征，DMA 通过人类反馈持续校准检索策略。

---

### 方案6：KVCOMM — 多智能体在线 KV-cache 通信（NeurIPS 2025）

**论文**：Ye et al. (2025) — *KVCOMM: Online Cross-context KV-cache Communication for Efficient LLM-based Multi-agent Systems*
**arXiv**: `2510.12872`（NeurIPS 2025 录用）

核心设计：**推理时在线共享 KV-cache**

- 多 Agent 之间不复制权重，而是**共享注意力状态**
- 在线协商哪些 KV-cache 该传递、该丢弃
- 降低 10x 通信开销

**和海马体的对应**：
- KV-cache 共享 ≈ 海马体的**记忆共享**（群体中多个个体共享经验）
- 在线协商 ≈ 海马体的**重放机制**（ replay，在睡眠/休息时重演记忆给其他脑区）

## 最优先方案：HiPPO Zoo + Memento 组合

**为什么这两个组合最优**：

```
HiPPO Zoo（底层 SSM）
    ↓ 联想记忆 + 自适应分配
    ↓ 存储在新场景的低能态
    ↓
Memento（上层案例记忆）
    ↓ 策略改进通过案例检索
    ↓ 不需要梯度更新
    ↓
Landauer 门（代价感知）
    ↓ 感知每次记忆检索的耗散
    ↓ 决定是否值得存储新案例
```

**为什么这个组合等效于海马体**：

| 海马体功能 | HiPPO Zoo + Memento 实现 |
|-----------|----------------------|
| 位置细胞（空间编码）| HiPPO 的正交多项式基 |
| 网格细胞（周期度量）| SSM 的循环状态更新 |
| 模式分离（DG 区）| HiPPO 的自适应分配 |
| 联想记忆（CA3）| Memento 的案例记忆检索 |
| 模式补全 | HiPPO 的联想记忆 |
| 输出到皮层（CA1）| Mamba 的 SSM 输出层 |

## 实验设计建议

**核心假设**：HiPPO Zoo + Memento 组合，在**新任务上的在线适应速度**，显著快于标准 Transformer。

**实验设计**：
1. **控制组**：标准 Mamba（或 Transformer）+ 固定 KV-cache
2. **实验组**：HiPPO Zoo + Memento 组合
3. **测量**：新任务出现后，第1/5/10次尝试的准确率提升曲线

**预期**：对照组的提升依赖梯度更新（慢），实验组的提升依赖记忆检索（快）。

**这直接验证了"海马体等效算法 vs 标准 Transformer"的在线学习差距。**
