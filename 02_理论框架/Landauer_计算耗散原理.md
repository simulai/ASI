# Landauer 计算耗散原理

> 来源：Rolf Landauer (1961), "Irreversibility and the Generation of Heat"
> 整理：2026-03-23

## 核心公式

```
E ≥ kT · ln 2 · 每bit擦除
```

每擦除1比特信息，必须耗散至少 kT·ln2 的能量（≈ 0.693 kT）

其中：
- k = Boltzmann常数
- T = 系统温度（开尔文）
- ln2 = 信息论与热力学的联系常数

## 物理意义

这是一个**不可逆性证明**：
- 擦除信息 = 不可逆操作 = 必须耗散能量
- 可逆操作（不擦除信息）理论上可以不耗散

## 对AI的意义

### 1. 计算有物理代价
每一步"计算"（无论是神经网络前向传播还是梯度下降）都是擦除信息的过程。
这意味着：
- 更大的模型 = 更多参数 = 更多bit擦除 = 更多能耗
- 但超过Landauer下限的能耗 = 无效做功 = 热浪费

### 2. 归纳偏置的热力学解释
为什么"硬归纳偏置"（Helix互补配对）有时比"软概率"更好？

Helix的互补配对 = **强制进入低能态** = 减少需要"搜索"的状态空间
= 减少bit擦除 = 更节能 = Landauer效率更高

### 3. 元认知的热力学触发
MetaGate = 曲率触发的自省 = 在"即将做大幅度状态跃迁"前停下
= 避免大幅bit擦除 = 节能 = 自组织临界性

### 4. 大模型 Scaling 的热力学极限
当模型参数量增长到一定程度，每一步前向传播的bit擦除总量会达到热力学极限。
这不是算法问题，是物理极限。
